# Forge Mobile Architecture
## Complete Feature Mapping: Desktop â†’ Mobile

---

## Executive Summary

This document maps every Forge desktop feature to its mobile equivalent, defining the new mobile-first architecture that will power the native Android app.

**Key Decisions:**
- **Navigation**: Bottom nav (4 tabs) + hamburger menu
- **Primary Interaction**: Chat-first (full-screen conversations)
- **Secondary Views**: Bottom sheets (not modals)
- **Gestures**: Swipe navigation throughout
- **Offline**: Core features work offline
- **Performance**: 60fps, <500KB bundle

## Summary

**Mobile Architecture Principles:**
1. **Chat-First** - Conversation is primary interaction
2. **Bottom-Up** - Bottom nav + bottom sheets
3. **Full-Screen** - No sidebars, maximize space
4. **Gesture-Rich** - Swipe, long-press, pull-to-refresh
5. **Offline-Capable** - Core features work offline
6. **Progressive** - Load fast, enhance progressively
7. **Native-Feeling** - Capacitor for native features
